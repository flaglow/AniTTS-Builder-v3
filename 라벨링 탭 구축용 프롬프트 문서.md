# 라벨링 탭 구축용 프롬프트 문서

## 목표

`clustering_slices.csv`를 기반으로, **라벨이 없는 클러스터를 반자동으로 추천 라벨링하고 사용자가 O/X 검수 후 확정**할 수 있는 **라벨링 탭(Labeling Tab)** 을 새로 만든다.

현재 CSV에는 다음 컬럼이 존재한다.

* `filename`
* `index`
* `timestamp_start`
* `timestamp_end`
* `transcript`
* `cluster_dir`

주의:

* 업로드된 `clustering_slices.csv`에는 **임베딩 벡터 컬럼이 없다.**
* 따라서 아래 기능에서 말하는 **클러스터 간 평균 유사도 계산**은, 별도의 임베딩 저장소 또는 오디오 조각으로부터 계산 가능한 임베딩 파이프라인이 이미 존재한다는 전제로 구현한다.
* `cluster_dir` 값에는 이미 사람이 읽을 수 있는 라벨(예: `하시다 이타루_2`)과 미라벨 클러스터(예: `clustering_74`)가 함께 존재한다.
* `noise`는 일반 라벨 후보에서 제외하는 것이 기본값이다.

---

## 구현 대상: 새 탭 `라벨링`

새로운 UI 탭 이름은 **라벨링**으로 한다.

이 탭의 목적은 다음 반복 루프를 빠르게 수행하는 것이다.

1. 자동 라벨 후보 추천
2. 샘플 청취 및 비교
3. 사용자의 O/X 판단
4. 확정 시 cluster_dir 갱신
5. CSV 리프래시 후 다음 클러스터 진행

---

## 핵심 UX 흐름

### 1. `자동 라벨` 버튼

사용자가 `자동 라벨` 버튼을 누르면, **라벨이 없는 클러스터 중 하나**를 선택해 자동 후보 추천을 수행한다.

라벨이 없는 클러스터의 기준:

* `cluster_dir`가 `clustering_숫자` 패턴인 항목
* `noise`는 제외 가능
* 이미 최종 라벨링 완료된 클러스터는 제외

추천 로직:

1. 현재 대상이 되는 **미라벨 클러스터 U**를 고른다.
2. 라벨이 존재하는 모든 클러스터 집합 **L**을 찾는다.

   * 예: `하시다 이타루_2`, `오카베 린타로_1` 등
   * 기본적으로 `noise`는 후보 라벨에서 제외
3. U에 속한 **모든 임베딩**과, 각 라벨 클러스터 L에 속한 **모든 임베딩** 사이의 유사도를 계산한다.
4. 각 라벨 클러스터별로 평균 유사도를 구한다.
5. 평균 유사도가 가장 높은 라벨 클러스터를 **추천 후보 1순위**로 선택한다.
6. 필요하면 상위 K개 후보(예: 3개)도 함께 내부적으로 유지할 수 있으나, UI에는 기본적으로 1순위 후보를 중심으로 보여준다.

유사도 정의:

* 기본값: cosine similarity
* 클러스터 간 평균 유사도:

  * `mean(sim(u_i, l_j)) for all i in U, j in L`
* 계산량이 큰 경우 다음 최적화 허용:

  * 임베딩 캐시 사용
  * 클러스터 centroid 우선 탐색 후 상위 후보에 대해서만 전체 평균 계산
  * 샘플 수 제한 옵션

### 2. UI에 보여줄 정보

`자동 라벨` 실행 후 UI에는 다음 두 영역을 보여준다.

#### A. 추천 라벨 후보 영역

표시 내용:

* **추천 라벨명**: 가장 평균 유사도가 높은 라벨 클러스터 이름
* **유사도 점수**: 평균 similarity score
* **추천 근거 샘플 3개**

  * 추천 라벨 클러스터에서 샘플 3개 표시
  * 각 샘플에 대해 다음 정보 표시

    * 클러스터 이름(라벨명)
    * transcript
    * filename 또는 원본 파일 식별자
    * timestamp_start ~ timestamp_end
    * 재생 버튼

#### B. 현재 미라벨 클러스터 영역

표시 내용:

* **현재 대상 클러스터명**: 예: `clustering_74`
* **샘플 5개** 표시

  * transcript
  * filename 또는 원본 파일 식별자
  * timestamp_start ~ timestamp_end
  * 재생 버튼
* 사용자는 각 샘플을 눌러 직접 들어볼 수 있어야 한다.

샘플 선택 원칙:

* 추천 라벨 쪽 3개 샘플은 대표성이 높은 순으로 선택

  * 예: 중심에 가까운 샘플, 길이가 너무 짧지 않은 샘플 우선
* 미라벨 클러스터 5개 샘플도 대표성 있게 선택

  * 가능하면 centroid 근처 + 다양성 확보

### 3. 사용자 판단 UI

사용자는 추천 결과를 보고 **유사 여부를 O / X로 판단**한다.

권장 UI 구성:

* `O (유사함)` 버튼
* `X (다름)` 버튼
* `직접 라벨 선택` 드롭다운 또는 검색형 셀렉트
* `건너뛰기` 버튼 (선택 사항)

동작:

#### O 선택 시

* 추천 라벨을 현재 미라벨 클러스터에 적용할 수 있게 한다.
* 적용 직전 확인 UI를 보여줄 수 있다.
* 확정 시 현재 미라벨 클러스터에 속한 모든 row의 `cluster_dir`를 추천 라벨명으로 변경한다.

#### X 선택 시

* 추천 라벨을 거절한다.
* 사용자는 다음 중 하나를 선택할 수 있어야 한다.

  1. 다른 기존 라벨을 수동 선택
  2. 새 라벨명 생성
  3. 보류 / 건너뛰기

#### 직접 라벨 선택 시

* 사용자는 기존 라벨 목록에서 선택할 수 있어야 한다.
* 검색 가능 UI 권장
* 새 라벨명을 직접 입력하는 옵션도 허용

---

### 4. 사용자 작업 완료 후 `Refresh cluster_dir`

사용자가 O/X 및 라벨 확정 작업을 완료하면, 기존에 존재하는 **`클러스터 디렉토리 리프래시 (Refresh cluster_dir)`** 버튼을 호출한다.

이 버튼의 역할:

* 최신 라벨링 결과를 반영해 `clustering_slices.csv`를 다시 생성하거나 덮어쓴다.
* 후속 반복에서 최신 `cluster_dir` 상태를 기준으로 추천 로직이 동작하게 만든다.

필수 요구사항:

* 라벨 확정 직후 메모리 상태만 갱신하지 말고, **실제 CSV 최신화**까지 반영
* refresh 완료 후 UI 데이터도 다시 로드

---

### 5. 반복 루프

위 과정을 반복한다.

반복 규칙:

1. 다음 미라벨 클러스터를 선택
2. 자동 라벨 추천
3. 샘플 비교 및 재생
4. O/X 또는 수동 라벨 확정
5. `Refresh cluster_dir`
6. 다시 1로 이동

우선순위 추천:

* 큰 클러스터부터 처리
* 또는 추천 confidence가 높은 클러스터부터 처리
* 또는 둘을 조합한 우선순위 점수 사용

예시 우선순위 점수:

* `priority = alpha * cluster_size + beta * confidence`

## 데이터 처리 규칙

### 입력 데이터

기본 입력은 `clustering_slices.csv`이다.

각 row는 최소한 아래 정보를 가진다.

* 원본 파일 경로 또는 식별자
* slice index
* 시작/끝 timestamp
* transcript
* 현재 cluster_dir

### 추가로 필요한 내부 데이터

자동 라벨 추천 기능에는 아래가 필요하다.

* 각 slice의 speaker embedding
* 또는 slice 오디오를 불러와 실시간/캐시 기반으로 embedding 생성 가능해야 함
* 각 샘플의 오디오 재생 경로 또는 로더

### 갱신 데이터

라벨이 확정되면:

* 대상 클러스터의 모든 row에 대해 `cluster_dir`를 새 라벨명으로 갱신
* refresh 버튼 호출 후 csv 반영

## 추천 알고리즘 상세 명세

### 기본 알고리즘

1. 미라벨 클러스터 U 선택
2. 라벨 클러스터 후보 집합 `{L1, L2, ...}` 구성
3. 각 `Li`에 대해:

   * `score_i = average cosine similarity between all embeddings in U and all embeddings in Li`
4. `argmax(score_i)`를 추천 라벨로 선택

### 성능 최적화 권장안

데이터가 큰 경우 전체 pairwise 계산 비용이 매우 크므로 다음 전략 권장:

#### 1차 후보 압축

* 각 클러스터 centroid를 미리 계산
* U centroid와 각 라벨 centroid 간 유사도로 상위 N개만 추림

#### 2차 정밀 계산

* 상위 N개 라벨에 대해서만 전체 샘플 평균 similarity 계산

#### 캐시

* cluster -> embeddings index 캐시
* cluster centroid 캐시
* 최근 추천 결과 캐시

## 예외 처리

### 추천 불가

다음 경우 자동 추천을 건너뛴다.

* 임베딩이 없음
* 라벨 클러스터가 너무 적음
* 유효 샘플 수 부족

이 경우 UI 메시지:

* `자동 추천을 생성할 수 없습니다. 수동 라벨링을 진행하세요.`

### confidence 낮음

최고 점수가 임계값 이하이면 경고 표시:

* `추천 신뢰도가 낮습니다. 샘플을 꼭 확인하세요.`

### 오디오 누락

샘플 재생이 불가능한 경우:

* 플레이 버튼 비활성화
* `오디오 없음` 표시

## 구현용 프롬프트 초안

아래 문장을 개발용 프롬프트 또는 작업 지시서로 사용할 수 있다.

> `clustering_slices.csv`를 기반으로 새 UI 탭 `라벨링`을 구현하라. 라벨이 없는 클러스터(`clustering_숫자`)를 대상으로 `자동 라벨` 버튼을 제공한다. 버튼을 누르면 해당 미라벨 클러스터의 모든 임베딩과, 라벨이 있는 각 클러스터의 모든 임베딩 간 cosine similarity를 계산하고, 클러스터별 평균 similarity가 가장 높은 라벨을 추천 후보로 선택하라. UI에는 추천 라벨 클러스터의 대표 샘플 3개와 라벨명(클러스터 이름), 현재 미라벨 클러스터의 대표 샘플 5개를 표시하고 각 샘플은 재생 버튼으로 들을 수 있어야 한다. 사용자는 O/X로 추천을 수락 또는 거절할 수 있어야 하며, 필요 시 기존 라벨 직접 선택 또는 새 라벨 생성도 가능해야 한다. 사용자가 라벨 확정 작업을 완료하면 기존 `Refresh cluster_dir` 버튼을 호출해 `clustering_slices.csv`의 `cluster_dir`를 최신화하고, 이후 다음 미라벨 클러스터에 대해 동일한 과정을 반복하라. `noise`는 기본적으로 추천 후보 라벨에서 제외하고, 임베딩이 없거나 추천 신뢰도가 낮은 경우 수동 라벨링으로 폴백하라.`

## 구현 전 체크리스트

* slice별 embedding을 어디서 가져오는지 확인
* 오디오 플레이 경로를 어떻게 구성하는지 확인
* `cluster_dir` 갱신 시 CSV write 경로 확인
* `Refresh cluster_dir` 버튼의 기존 동작 확인
* `noise` 처리 정책 확정
* 새 라벨 생성 시 naming rule 확정

---

## 추가 권장 사항

* 추천 점수 상위 3개 라벨도 접을 수 있는 UI로 제공하면 검수 효율이 올라간다.
* 큰 클러스터 우선 처리 기능을 넣으면 전체 라벨링 속도가 빨라진다.
* 확정 이력 로그를 남기면 롤백과 QA가 쉬워진다.
* 대표 샘플 선택은 무작위보다 centroid 기반이 더 안정적이다.
